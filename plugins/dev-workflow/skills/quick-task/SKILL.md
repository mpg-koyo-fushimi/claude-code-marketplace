# Quick Task スキル

やることが明確なタスクを、最小限の探索と低推論でテンポ良く遂行するためのモード。

## 適用判断

以下を満たすときに使用する:
- 変更範囲と意図が明確
- 設計上の分岐判断が不要
- 影響範囲が限定的（広範囲の調査・リファクタ不要）

曖昧さ・設計判断・広範な影響が見込まれる場合は `deep-task` を使うこと。

## 実行方針

1. 不要な探索を省き、対象ファイルへ直行する
2. 既存の規約・スタイルに合わせる（周辺コードを最小限確認）
3. 過剰な抽象化・リファクタ・コメント追加をしない
4. 完了したら変更内容を1〜2文で要約する

## やらないこと

- 求められていないリファクタ
- 推測に基づく機能追加
- 冗長な検証ステップ
