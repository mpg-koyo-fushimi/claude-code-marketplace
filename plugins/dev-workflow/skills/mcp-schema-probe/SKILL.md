# MCP スキーマ棚卸しスキル

新しい MCP サーバを使う前や、既存 MCP で「この操作できたっけ？」を確認したいときに、スキーマレベルでツール一覧と制約を可視化する。

## 適用判断

以下のタイミングで使用する:
- 新しい MCP 連携を組み込む前に実現可能性を確認したい
- 複数 MCP サーバを横断して比較したい
- skill 設計中に「このサービスで日付フィルタはできるか？」など特定の観点を素早く確認したい

**やらないこと**: 実際にツールを呼んでデータを取得する調査はこのスキルでは行わない（あくまでスキーマ棚卸し）。

## 引数

- `<mcp-server-name or keyword>`（必須）: 調査対象のサーバ名またはキーワード（例: `slack`, `notion`, `figma`）。カンマ区切りで複数指定可
- `[observation-question...]`（任意）: 観点を 1 つ以上の質問で渡す（例: `"ユーザーで絞り込めるか"`, `"日付フィルタがあるか"`, `"書き込みできるか"`）。省略時は汎用観点（読み込み/書き込み/検索/フィルタリング）を使用

## ステップ

### Step 1: ツール一覧の取得

`$ARGUMENTS` を解析し、サーバ名とキーワードを抽出する。

対象ごとに `ToolSearch` を実行し、関連ツールを列挙する:

```
ToolSearch(query: "<server-name>", max_results: 10)
```

複数サーバを指定した場合は並列実行する。

### Step 2: スキーマの一括ロード

Step 1 で得たツール名を `select:` 形式でまとめてロードする:

```
ToolSearch(query: "select:<tool1>,<tool2>,<tool3>,...", max_results: <N>)
```

### Step 3: 観点マトリクスの作成

引数で渡された観点（または省略時は下記の汎用観点）に照らして、各ツールを評価する。

**汎用観点（引数省略時のデフォルト）**:
- 検索・一覧取得ができるか
- ユーザー/メンバーで絞り込めるか
- 日付・期間で絞り込めるか
- データを作成・更新できるか（書き込み系）
- ページネーション対応か

各観点について以下のマークで評価する:
- ✅ 対応（パラメータが存在する）
- ❌ 非対応（パラメータがなく、スキーマ上不可）
- ⚠️ 条件付き（別の方法で代替可能、または制約あり）

### Step 4: 結果の報告

以下の形式で出力する:

```markdown
## <サービス名> MCP スキーマ棚卸し

### ツール一覧と観点マトリクス

| ツール名 | 機能概要 | <観点1> | <観点2> | ... | 制約メモ |
|---|---|---|---|---|---|
| tool_name | 1行説明 | ✅ | ❌ | ... | 備考 |

### 判明した制約・注意点

- <制約1>（例: ユーザーアクティビティの取得は非対応）
- <制約2>

### 利用可能な代替手段

- <観点X> について: <代替ツール名> で <方法>
```

## 並列調査のヒント

複数の MCP サーバを同時調査する場合は、サーバごとに Explore サブエージェントに分散させると高速に処理できる。その場合は各エージェントにサーバ名と観点を同一の形式で渡し、結果をまとめてマージする。

## エラーハンドリング

| シナリオ | 対応 |
|---|---|
| ToolSearch でツールが 0 件 | 「対象サーバは利用可能な MCP に含まれていません」と報告して終了 |
| 引数のサーバ名が曖昧で複数マッチ | マッチしたツール名を全部列挙し、ユーザーに絞り込みを確認 |
| スキーマに説明文がない | `description` 未記載として ⚠️ でマーク、パラメータ一覧のみ記載 |
