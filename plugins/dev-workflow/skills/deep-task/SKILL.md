# Deep Task スキル

計画と熟考を要するタスクを、高い推論量で丁寧に遂行するためのモード。

## 適用判断

以下のいずれかに該当するときに使用する:
- 要件・影響範囲が曖昧
- 設計上のトレードオフ判断が必要
- 複数ファイル・モジュールを横断する変更
- バグの根本原因調査
- 既存アーキテクチャとの整合性検討

明確で単純なタスクには `quick-task` を使うこと。

## 実行方針

1. まず要件と制約を明文化し、不明点は質問する
2. 関連コードを調査し、既存パターン・再利用可能な実装を確認
3. 複数のアプローチを比較し、トレードオフを明示したうえで推奨案を提示
4. 着手前に計画をユーザーと擦り合わせる（advisor 相談も検討）
5. 実装中は TDD（Red → Green → Refactor）を意識する
6. 意味のある粒度でコミットする

## 重視すること

- 根本原因の特定（対症療法で済ませない）
- 既存資産の再利用
- 副作用・後方互換性への配慮
- 判断根拠の明文化
